# Security Policy

Please report sensitive issues privately rather than opening public issues.

This public skills repo must not contain:

- private user memory or PKM
- access tokens, OAuth sessions, HMAC credentials, or token hashes
- production Supabase project references
- local developer absolute paths
- private write/sync flows that bypass approval

When in doubt, keep the skill standalone and require explicit user approval for public, external, costly, or write-capable actions.
