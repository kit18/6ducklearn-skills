# Methodology Catalog

Use this catalog when the user asks what methods exist, wants source anchors, or needs a buying/learning path. The skill routes across three layers: framework library, consultant problem-solving, and strategy-to-deck expression.

## Practical Routing Hierarchy

1. If the audience must approve a recommendation, start with Pyramid Principle.
2. If the answer is uncertain, start with McKinsey Mind, McKinsey problem classification, or Bulletproof Problem Solving.
3. If the user asks for SWOT or a model, use Key Management Models to pick the framework, then turn it into a decision.
4. If the question is about competition, use Porter/Magretta and HBR strategy.
5. If the question is about where to compete and how to win, use Playing to Win.
6. If the question is about a new venture, product strategy, monetization, or GTM model, use Business Model Generation.
7. If the user needs reusable consulting PPT format, use Modular Presentation as the deck assembly and health-check layer.
8. If the output needs charts, use Say It With Charts and data storytelling as support, not as the core storyline.

## Recommended Buying Order

1. The Pyramid Principle.
2. Key Management Models.
3. HBR's 10 Must Reads on Strategy.
4. The McKinsey Mind / 麥肯錫的專業思維 or Bulletproof Problem Solving.
5. Business Model Generation.
6. Playing to Win.
7. Say It With Charts.
8. Modular Presentation / 我用模組化簡報，解決99.9%的工作難題.

For Chinese-language consulting PPT preparation, add 麥肯錫問題分析與解決技巧 when the user needs problem classification, issue conversion, alternatives, and scenario analysis before slide planning.

For the most practical "I need to make strategy PPTs" stack, start with: The Pyramid Principle, Key Management Models, HBR's 10 Must Reads on Strategy, The McKinsey Mind / 麥肯錫的專業思維, Say It With Charts, and Modular Presentation for reusable PPT section assembly.
