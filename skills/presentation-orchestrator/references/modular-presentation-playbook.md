# Modular Presentation Playbook

Use this reference when the user asks for consulting PPT format, reusable slide modules, Chinese/Taiwan business presentation style, or a faster way to turn rough work notes into a polished deck.

This is a synthesized reference inspired by public descriptions and reviews of Liu Yiyou's book *我用模組化簡報，解決99.9%的工作難題：簡報職人教你讓全球頂尖企業都買單的企業簡報術*. Do not copy book text. Use the source as a practical anchor for modular thinking: clarify the job, decompose content into reusable modules, translate information into visual insight, and health-check the deck.

## Consulting Adaptation

Use modular presentation thinking as the PPT-format layer under the consulting storyline:

1. Clarify purpose and audience.
   - What decision, belief, or action should change?
   - Who is listening, and what do they already care about?
2. Choose modules, not a fixed template.
   - Select only the deck modules needed for the decision.
   - Reorder modules based on audience familiarity and risk.
3. Build the three-part arc.
   - Open: context, tension, decision ask, or recommendation.
   - Body: logic framework, evidence, options, tradeoffs.
   - Close: decision, action, owner, next step.
4. Convert content into slide modules.
   - Each module has one purpose, one main claim, one proof type, and one visual pattern.
5. Health-check the deck.
   - Remove content that does not support the decision.
   - Mark weak evidence instead of filling the gap with invented certainty.

## Consulting PPT Module Library

| Module | Use when | Slide archetypes | Key question |
|---|---|---|---|
| Decision brief | The audience must approve, fund, stop, or align | title with ask, executive summary, decision memo | What exactly must they decide? |
| Audience context | The audience is not yet aligned on the situation | SCQA opener, situation/tension, stakeholder map | What do they already believe, and what must shift? |
| Route card | The business question is messy | route card, decision-state map, method selection | What kind of problem is this? |
| Consulting POV | The deck needs a point of view | recommendation card, hypothesis card, governing thought | What is the answer-first message? |
| Structured diagnosis | The cause or opportunity is unclear | issue tree, SWOT/TOWS, Five Forces, value chain, canvas | What structure explains the situation? |
| Evidence ledger | Claims need proof discipline | claim/evidence/gap table, source matrix, appendix map | What proves each claim, and what is still a gap? |
| Options | Leaders must choose among paths | 3-option comparison, decision table, value/risk matrix | Which paths were considered and rejected? |
| Recommendation | The deck must land on one route | recommendation summary, why-this-now, tradeoff card | Why this option now? |
| Execution | Approval requires feasibility | roadmap, owner/action table, milestone plan, test gates | What changes on Monday? |
| Risk and objection | Decision risk is material | risk matrix, red-team table, objection/response slide | What will skeptics ask? |
| Decision close | The audience must act after the meeting | final ask, next-step checklist, approval request | What must be approved before people leave? |
| Appendix | Details may be challenged | data appendix, methodology notes, backup analysis | What proof should be ready but not in the main flow? |

## Recommended Consulting PPT Format

Use this default sequence for executive consulting decks:

1. Decision ask or answer-first title.
2. Executive summary: recommendation, 3 reasons, risk, ask.
3. Situation and tension: why this matters now.
4. Route card: decision state and selected method.
5. Consulting POV: recommendation or hypothesis.
6. Structured diagnosis: issue tree or chosen business framework.
7. Evidence ledger: key claims, source types, and gaps.
8. Options considered: usually three paths when a choice is needed.
9. Recommendation and tradeoffs.
10. Execution plan: owner, timeline, test gates, Monday actions.
11. Risks, objections, and mitigations.
12. Final decision ask.
13. Appendix evidence.

Shorten aggressively for senior audiences. If the recommendation is already known, start with the answer and move context to backup.

## Visual Translation Rules

- Turn data into insight before choosing the chart.
- Use action titles; topic titles hide the point.
- Use one main visual per slide: chart, table, diagram, roadmap, or canvas.
- Use grouping, alignment, contrast, whitespace, and repetition to make the module scannable.
- Use three options only when it clarifies a choice; do not force trios into every deck.
- Translate text-heavy source notes into: point, logic, evidence, implication, action.

## Health Check

Before final output, ask:

- Purpose: does the deck solve a business problem or ask for a decision?
- Audience: is the story written for this audience, not a generic reader?
- Logic: can the main argument be explained in 3 to 5 sections?
- Module fit: does every module have a clear job?
- Evidence: does every major claim have a source type or explicit gap?
- Visuals: can each slide be understood in 5 seconds?
- Action: can the audience identify the requested decision and next step?

Fail and revise if the deck is a pile of information, uses frameworks as decoration, or makes the audience search for the point.
