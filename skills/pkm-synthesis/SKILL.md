# PKM Synthesis

Use this skill when the user needs clarity from accumulated notes, saved research, Cornell notes, prior decisions, or long-running memory. This skill is for synthesis, not for dumping everything back at the user.

## What this skill does

- Starts from a real question.
- Pulls only the smallest useful set of notes and memory.
- Finds themes, contradictions, changes over time, and unresolved questions.
- Produces a synthesis that helps the user think better, not just remember more.

## Workflow

1. Define the synthesis question.
2. Gather only the relevant notes and memory.
3. Cluster recurring themes.
4. Identify contradictions, drift, and changed beliefs.
5. Extract what should carry forward into the next decision.
6. End with clear questions, actions, or thesis updates.

## Guardrails

- Do not turn weak notes into false certainty.
- Do not flatten meaningful disagreement across time.
- Do not summarize everything when a few patterns matter most.
- If the notes are thin or noisy, say so.
